# 09 - Benchmarking and Observability

## Scope

- Build a disciplined measurement surface for TTFT, ITL, TPS, RPS, utilization, memory, and traces.
- Use benchmark harnesses and dashboards to make runtime claims defensible.

## Notes

Use `notes/` for benchmark methodology, AIPerf or MLPerf discipline, DCGM metrics, host-side counters, and trace interpretation.

## Experiments

Use `experiments/` for load tests, dashboard snapshots, trace collection workflows, and repeatable benchmark suites.

## Exit Criteria

- Every experiment yields percentiles, throughput, utilization, memory behavior, and a short diagnosis.
- Benchmark results are repeatable enough to support decisions.
